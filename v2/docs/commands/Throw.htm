Throw

发生错误时发出信号. 此信号可以被 try-catch 语句捕获.

Throw Value

参数

Value

要抛出的值; 通常是一个 Error 对象. 例如:

throw ValueError.new("Parameter #1 invalid", -1, theBadParam)

由于此参数为表达式, 所以下面所有的例子也都是有效的:

throw 3
throw "literal string"
throw MyVar
throw i + 1
throw { what: "Custom error", file: A_LineFile, line: A_LineNumber } ; 抛出一个对象

如果省略, 则抛出带默认消息的 Error.

备注

如果表达式被括在括号中, 则 throw 后的空格或制表符是可选的 , 如 throw(Error.new()) 所示.

如果使用了 Try/Catch, 则 Value 赋值到 Catch 的 OutputVar.

如果没有使用 Try(或与 Finally 一起使用, 但没有 Catch), 将发生以下情况:

  1. 任何激活的 OnError 回调都会被调用. 每个回调可以检查 Value 并抑制或允许进一步的回调和默认处理.
  2. 默认情况下, 会根据抛出的内容显示错误信息. 如果 Value 是一个对象 并且拥有一个名为 Message 的值属性, 那么它的值将被用作消息. 如果 Value 是一个非数字字符串, 则使用它作为消息. 在任何其他情况下, 都会使用一个默认的消息. 如果 Value 是数字, 则显示在默认消息的下方.
