StringCaseSense

确定字符串比较是否区分大小写(默认为 "不区分大小写").

StringCaseSense OnOffLocale

参数

OnOffLocale

类型: 字符串或整数(布尔值)

下列值之一:

On 或 1(true): 字符串比较时区分大小写. 此设置也让表达式等号运算符(=) 和 InStr(不区分大小写模式) 使用下面的 locale 方式进行处理.

Off 或 0(false): 字母 A-Z 被视为等同于它们的小写字母. 为了向后兼容和保持高效, 这是所有脚本的初始默认值(根据被比较字符串的性质, 使用 Locale 比 Off 慢 1 到 8 倍).

Locale: 根据当前用户的区域设置规则, 本地字符串比较不区分大小写. 例如, 在大多数英语及西欧地区, 不仅将 A-Z 视为等同于它们的小写形式, 同时也将 ANSI 字母(如 Ä 和 Ü) 视为等同的.

备注

此设置适用于:

  • 表达式比较运算符(== 除外). 但是, 由于等号运算符(=) 总是不区分大小写, 当 StringCaseSense 设置为 On 时, 它使用 Locale 模式, 与 InStr 的不区分大小写模式相同.
  • InStr. 然而, InStr 在其 CaseSensitive 参数为 true 时, 不会受此设置影响.
  • StrReplace

内置变量 A_StringCaseSense 包含了当前设置(单词 On, Off 或 Locale).

每个新运行的线程(如 热键, 自定义菜单项或定时子程序) 都会以此函数的默认设置开始. 这个默认设置可以通过在脚本的自动执行段(脚本的顶部) 使用此函数来改变.

相关

StrReplace, InStr

示例

#1

StringCaseSense "Locale"
